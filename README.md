# Сервис прогнозирования дефолта по кредитным картам
Проект посвящён разработке и внедрению ML‑сервиса для прогнозирования дефолта клиентов по кредитным картам.  
В качестве датасета используется `UCI_Credit_Card.csv` из набора **Default of Credit Card Clients**.  
Цель проекта — показать полный цикл работы с моделью: анализ данных, обучение, сохранение модели, запуск API и контейнеризация.

Используются данные о клиентах кредитной карты, их платёжной истории и финансовом поведении.  
Целевая переменная — `default.payment.next.month`, где:

- `1` — дефолт в следующем месяце;
- `0` — дефолта нет.

Основные группы признаков:
- демографические;
- финансовые;
- история платежей;
- суммы счетов и платежей.

## Модель
Для задачи используется простая модель бинарной классификации.  
В качестве базовой модели выбрана **LogisticRegression**.  
Обученная модель сохранена в файл `models/.joblib.

## API
Сервис реализован на Flask и содержит два эндпоинта:

### `GET /health`
Проверка работоспособности сервиса.

```json
{
  "status": "ok",
  "model_loaded": true
}
```

### `POST /predict`
Принимает JSON с признаками клиента и возвращает прогноз и вероятность дефолта.

```json
{
  "prediction": 0,
  "probability": 0.4736084378720057
}
```

## Локальный запуск

### 1. Создать виртуальное окружение
```bash
python -m venv venv
```

### 2. Активировать окружение
**Linux / macOS:**
```bash
source venv/bin/activate
```

**Windows:**
```bash
venv\Scripts\activate
```

### 3. Установить зависимости
```bash
pip install -r requirements.txt
```

### 4. Запустить сервис
```bash
python app/api.py
```

Сервис будет доступен по адресу `http://localhost:5000`.

### Проверка healthcheck
```bash
curl http://localhost:5000/health
```

### Пример запроса на предсказание
```bash
curl -X POST http://localhost:5000/predict \
-H "Content-Type: application/json" \
-d '{ ... }'
```

## Запуск в Docker
Сервис можно запустить в контейнере через `docker-compose.yml`.

### Сборка
```bash
docker compose build
```

### Запуск
```bash
docker compose up
```

Если нужен только образ:
```bash
docker build -t credit-default-api .
docker run -p 5000:5000 credit-default-api
```

## A/B-тестирование
Для сравнения версий модели можно провести A/B‑тест:

- контрольная группа — `v1`;
- тестовая группа — `v2`;
- распределение трафика — случайное, например 50/50.

### Метрики
- основная: `F1-score`;
- дополнительная: `Precision` или `Recall`.

## Архитектура сервиса
В проекте используется монолитная архитектура: модель ML и API объединены в одном Flask‑приложении. Этот подход снижает порог входа и упрощает отладку. В дальнейшем возможна миграция на микросервисы.

### Оркестрация и Docker Compose
Для развёртывания применяется docker-compose.yml — он автоматизирует запуск сервиса в контейнере. Опционально можно подключить Nginx в роли reverse proxy. Такой подход демонстрирует базовый принцип оркестрации и обеспечивает воспроизводимость окружения.

### Логирование и мониторинг
Сервис формирует JSON‑логи по каждому запросу: фиксирует timestamp, endpoint, входные данные, прогноз, вероятность и статус. В production-среде логи агрегируют в централизованной системе (например, ELK) для анализа ошибок и нагрузки.

### DVC и MLflow
DVC обеспечивает версионирование данных и ML‑артефактов, MLflow — трекинг экспериментов (параметры, метрики, результаты). Инструменты иллюстрируют ключевые практики MLOps, актуальные для управления жизненным циклом моделей.

### Бизнес‑метрики
Помимо ML‑метрик, оцениваются финансово‑ориентированные показатели: ожидаемые потери (PD × сумма кредита), доля одобренных заявок при заданном уровне риска. Это позволяет соотнести качество модели с бизнес‑целями кредитной организации.
